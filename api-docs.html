Overview

Introduction

This document describes all available API endpoints for the Bills Bike Tour backend. The API is hosted on Cloudflare Workers and is used exclusively by the Admin Panel to manage users and renew tour access.

Base URL

https://bike-tour-backend.billsbiketour.workers.dev

Authentication

Every request must include the header: x-admin-key: <secret>
Requests missing this header will be rejected with 401 Unauthorized.

Endpoints

API Reference

GET /admin/users List All Users

Returns a paginated list of all registered users on the platform. Results can be filtered by status using an optional query parameter. Each user object includes their email, current status, join date, and expiry date.

Query Parameters
Parameter Type Required Description
status string Optional Filter users by status. Accepted values: active or expired. Omit to return all users.
Example Requests
# All users
GET /admin/users
x-admin-key: <secret>

# Active users only
GET /admin/users?status=active
x-admin-key: <secret>

# Expired users only
GET /admin/users?status=expired
x-admin-key: <secret>
Response Body
{
  "total": 2,
  "users": [
    {
      "email":       "alice@example.com",
      "status":      "active",
      "created_at":  "2026-04-11 10:00:00",
      "expires_at":  "2026-04-13 10:00:00"
    },
    {
      "email":       "bob@example.com",
      "status":      "expired",
      "created_at":  "2026-03-01 10:00:00",
      "expires_at":  "2026-03-08 10:00:00"
    }
  ]
}

POST /admin/renew-access Renew User Access

Generates a new JWT token for the specified user and sends it to their email address. This restores access to their expired tour. The token is delivered via email — the user does not need to take any action on the admin panel.

Request Body (JSON)
FieldTypeRequiredDescription
email string Required Email address of the user whose access should be renewed.
Example Request
POST /admin/renew-access
Content-Type: application/json
x-admin-key:    <secret>

{
  "email": "alice@example.com"
}
Response Body
# Success
{
  "message": "Access renewed and email sent successfully."
}

# Error
{
  "error": "User not found."
}
Status Codes
200
Token sent to user's email
401
Missing or invalid admin key
500
Failed to generate or send token
GET /admin/flags Get Feature Flags

Returns the current state of all feature flags. Use this to check whether payment is currently required before allowing users to access the tour.

Example Request
GET /admin/flags
x-admin-key: <secret>
Response Body
{
  "payments_enabled": true
}
Response Fields
FieldTypeDescription
payments_enabled boolean Whether payment is currently required to access the tour.
Status Codes
200
Success — flags returned
401
Missing or invalid admin key
500
Internal server error
PUT /admin/flags Update Feature Flags

Updates one or more feature flags. Send only the fields you want to change. Returns the full updated flag state after applying the changes.

Request Body (JSON)
FieldTypeRequiredDescription
payments_enabled boolean Required Set to true to require payment, false to disable it.
Example Request
PUT /admin/flags
Content-Type: application/json
x-admin-key:    <secret>

{
  "payments_enabled": false
}
Response Body
{
  "payments_enabled": false
}
Status Codes
200
Success — flags updated
401
Missing or invalid admin key
500
Internal server error
Reference

Data Formats

FieldFormatNotes
created_at YYYY-MM-DD HH:MM:SS UTC timestamp. No timezone suffix. Treat as UTC.
expires_at YYYY-MM-DD HH:MM:SS UTC timestamp. If this value is in the past, user status is expired.
status "active" or "expired" Computed server-side from expires_at vs. current time.
email string Unique identifier for a user. Used as the primary key across all endpoints.
